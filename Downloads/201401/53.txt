I recently learned how to define multiple profiles in my pom.xml files, and
activate certain ones by name in my ~/.m2/settings.xml. This helps me with
complex Maven-Thrift builds, where I want a `mvn generate-sources` locally,
but my manager doesn't want the remote, continuous integration build to
require the thrift compiler. For the most part, this works wonderfully.

However, I also work on Maven projects that have nothing to do with Thrift,
so their pom.xml files don't define a "thrift" profile. For these projects,
when I run Maven commands (e.g. `mvn test`), Maven spits out a warning:

Maven attempts to activate a "thrift" profile, but silently ignores it when
missing. Is such a configuration possible?
