Hi Gerrit,

This is an old topic, and you may have solved it some way or another by =
now.=20
But just as I struggled with this same problem, I figured I would share =
the answer for the benefit of others running into this.

The problem is that 64-bit Windows has two system profile folders. =
32-bit processes get redirected to =
%SystemRoot%\SysWOW64\config\systemprofile when accessing =
%SystemRoot%\System32\config\systemprofile.

When you copied the .ssh\known_hosts file with Windows Explorer, you put =
it in the folder where only other 64-bit processes would see it.  You =
are likely running maven with a 32-bit Java and it is expecting to find =
the .ssh/ folder under the SysWOW64 system profile path.
