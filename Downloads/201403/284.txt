I made an external dependency of servlet 2.5 earlier version for the
standard taglib in my app.
This taglib had a transitive dependency on an earlier version of the
servlet api that was messing things up.
By removing this dependency from the path, it worked.
if you do mvn dependency:tree you will see which jar it is that is dragging
in the earlier version of the servlet spec.
