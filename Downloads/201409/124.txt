We do this extensively with maven, using the maven release plugin.

Our artifacts are in turn installable packages (rpms in our case, but could b=
e deb, whatever) which get published into a yum repository.

The code is "promoted" from dev to test to higher environments by hard linki=
ng the packages into distinct yum repos, a yum repo for each environment.

The release created by the release plugin is built just once and promoted. T=
he rpms are created using the rpm-maven-plugin.
